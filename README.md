# kvittal-personal-work

Created a sample monolith application that explains a brief process of vehicle order management.

In an ideal scenarion, when a dealer visits the application, he/she will be placing either a vehicle order or contract an existing available vehicle to a customer as part of vehicle order management.

In the attached code, both these sceanrios are handled within a monolith app that solely relies on a single app server and in case of any changes, we may experience downtime.

As part of a futuristic state, we propose to break these functionalities into individual services so that changes specific to a functionality, need not impact the wider application and inturn results in lesser downtime.
